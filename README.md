# SourceMate

SourceMate - учебный веб-сервис и Telegram-бот для поиска научных источников, подбора релевантных публикаций и проверки учебных работ на возможные заимствования.

Проект помогает студентам, преподавателям и исследователям быстрее находить материалы по теме, оценивать их релевантность и получать структурированный отчет по загруженному документу.

## Возможности

- Поиск научных источников по пользовательскому запросу.
- Агрегация данных из Crossref, Semantic Scholar, Wikipedia, OpenAlex, CyberLeninka и НЭБ.
- Интеллектуальное расширение запроса и оценка релевантности через локальную LLM Ollama.
- Telegram-бот с постраничной выдачей результатов.
- Веб-интерфейс с главной страницей, авторизацией и личным кабинетом.
- Регистрация, вход, восстановление доступа через код на почту.
- Профиль пользователя, загрузка аватара, смена пароля и настройки аккаунта.
- Загрузка TXT, DOCX и PDF-документов для проверки.
- Расчет оригинальности, совпадений и списка похожих источников.
- Детальная страница отчета с проблемными фрагментами и рекомендациями.
- Карточка источника с DOI, авторами, ссылкой, ГОСТ- и BibTeX-оформлением.

## Установка

```bash
git clone https://github.com/ivlieva006/SoureMate.git
cd SoureMate
npm install
```

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Минимальные переменные для Telegram-бота:

```env
BOT_TOKEN=telegram_bot_token
LLM_ENABLED=true
LLM_BASE_URL=http://localhost:11434/v1
LLM_MODEL=llama3.1
```

Для веб-версии:

```env
WEB_PORT=3000
MAIL_FROM=SourceMate <user@example.com>
MAIL_DEV_FALLBACK=true
```

Для реальной отправки писем можно настроить Brevo:

```env
BREVO_API_KEY=your_brevo_api_key
```

или SMTP:

```env
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=user@example.com
SMTP_PASS=app-password
SMTP_FROM=SourceMate <user@example.com>
```

## Как работает проверка документов

1. Пользователь загружает TXT, DOCX или PDF-файл в личном кабинете.
2. Сервер извлекает текст из документа.
3. SourceMate определяет тему проверки или использует тему, указанную пользователем.
4. Агрегатор ищет релевантные источники.
5. Система загружает доступный текст найденных источников.
6. Документ сравнивается с источниками по шинглам.
7. Формируется отчет: оригинальность, совпадения, похожие источники, фрагменты и рекомендации.

## Основные API

- `GET /api/auth/me` - текущий пользователь.
- `GET /api/cabinet/state` - состояние личного кабинета, сессии и проверки.
- `POST /api/auth/register/request` - запрос кода регистрации.
- `POST /api/auth/register/verify` - подтверждение регистрации.
- `POST /api/auth/login` - вход.
- `POST /api/auth/recover/request` - запрос кода восстановления.
- `POST /api/auth/recover/verify` - подтверждение кода восстановления.
- `POST /api/auth/recover/reset` - установка нового пароля.
- `POST /api/account/profile` - обновление профиля.
- `POST /api/account/password` - смена пароля.
- `POST /api/account/avatar` - загрузка аватара.
- `POST /api/antiplagiarism/check` - проверка документа.
- `POST /api/cabinet/check/delete` - удаление проверки.

## Данные и безопасность

- `.env` не коммитится в репозиторий.
- Локальное runtime-хранилище находится в `data/users.json` и не коммитится.
- Пароли хранятся в виде PBKDF2-хеша с солью.
- Сессии передаются через HTTP-only cookie.
- Для локальной разработки можно включить `MAIL_DEV_FALLBACK=true`, чтобы видеть код подтверждения без настроенной почты.

## Пример сценария

1. Запустить Ollama.
2. Запустить `npm run web`.
3. Зарегистрироваться на странице `/auth.html`.
4. Открыть личный кабинет.
5. Загрузить документ и указать тему.
6. Получить отчет по оригинальности и открыть найденные источники.

## Статус проекта

Проект является учебным MVP. Он уже включает поиск источников, веб-кабинет, авторизацию, загрузку документов и формирование отчетов. В дальнейшем можно подключить полноценную базу данных, расширить экспорт отчетов, улучшить алгоритмы проверки и добавить интеграции с образовательными платформами.
